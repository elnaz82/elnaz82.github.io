Trial Download Redesign

Registration page to start trial version of Quest products

Feb 2018 - Mar 2019

Deliverables

  • Competitive analysis
  • User flow diagram for the form
  • Pixel perfect mockups (Desktop, Tablet, Mobile)

Goals

  • Research best practiced for online forms
  • Dicsover exsiting issues using analytics
  • Eliminate extra fields
  • Improve form validation
  • Increase the speed of filling out the form

Use Cases

  • As a user, I want to provide minimal information to start using the trial.
  • As a user, I expect to not fill out a form when I'm logged in (existing customer) so that I don't have to provide my information again.
  • As user, I need to clearly see the required fields so that I can skip optional items.
  • As user, I need to fix the errors during filling out the form as soon as they occur so that I can fix them immediately before I forget.
  • As a user, I need to get feedback about my successful form submission so that I can start using my trial after all the time I spent on the form.

Features

  • Summarize product benefits & key features
  • Start with a short one field form (new users)
  • Accept terms and conditions and download right away (existing user)
  • Auto fill items based on existing data
  • Live inline validation
  • Clear form labels
  • Receive the product key and files via email

Form Research

I started by reading several articles and research papers to find out the primary rules for designing an efficient form. Followings are some of the best practices I captured:

  • Explicitly mark both required and optional fields
  • Live inline validation has seen an explosive growth in adoption
  • The shorter the form, the better
  • Avoid defaulting fields to certain values unless you believe a large portion of your users will select that value
  • Use input masks where necessary
  • Auto-focus first form field in desktop to create an indication of where to start
  • Match keyboard to input type in mobile
  • Use autocomplete where possible to limit typing
  • Use sentence case for labels as they are easier to read
  • Top-aligned labels have the fastest completion time
  • Maintain inline label's placeholder text on focus. (Floating labels is the way to go in this case)
  • Avoid generic words for submit button
  • Avoid reset button and use the main CTA in the place that has the most convenient position for clicking in the viewport

Form User Flow

Based on my findings and working with the business analyst about the requirements of the flow I have come up with the following flow/style for the new form:

  • A short form to capture users email address. If the user is existing, no more information will be required, otherwise, the second form will be presented to capture additional information
  • Inline floaty labels to solve the issue with international localities text labels and improve the speed of filling out the forms
  • Required and optional fields are identifiable through *
  • Visual indicators for interactions (blue for focus, red for errors, green check marks for complete) to make the form more intuitive
  • Inline live validation to:
    • make locating errors easier
    • improve the speed of fixing errors while the field context is fresh in users minds
    • avoid skipping a required field unintentionally
  • Sentence case labels since they are easier to read
  • Centered aligned orange CTA button for larger touch area (mobile friendly)
