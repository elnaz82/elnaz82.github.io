Product Pages Redesign

Mar 2018 |

Project Overview

The purpose of this project were to:

  • make product pages more dynamic and flexible to meet business needs
  • create unique identity (theme) for individual business units
  • highlight key features of products to help improve user engagement
  • conduct competitive analysis and add any missing information to the page
  • align style with Quest branding
  • set up a split test to measure success
  • implement improvements based on test results

My Role

  • Design themes for 2 different business units using product specific design patterns
  • Redesign sticky tabs
  • Design different options for hero space
  • Design a section to showcase product benefits
  • Design different ideas to display key features of the product
  • Redesign product tour and improve interaction
  • Redesign product resources and develop a prototype to show functionality by hand coding Html, CSS3
  • Design a ribbon to show a quote from our customers (reviews)
  • Showcase customer logos
  • Design a ribbon to show statistics
  • Design a ribbon to display awards and recognitions of the product
  • Redesign social share
  • Conduct competitive analysis for any new component (Quote, Stats, Awards, Benefits, Hero, Tour, etc)

Issues

  • Text heavy layout
  • Key features of the products were buried in text and selling points can be easily missed by users
  • Lack of flexibility to modify layout and add new sections
  • Lack of interactivity on page components
  • Not enough information to support benefits of using the product
  • Same theme for all products

Solution

  • Visualize data using images or icons and get rid of heavy text to increase users focus
  • Dynamic template concept to enable business units create flexible content
  • Introduce new patterns to increase engagement and user experience
  • Add existing data buried in other resources to the page to support benefits (Quote, Awards, Stats, etc)
  • Design different themes based on product and give unique look to each business unit

Challenges

  • Updating product page template means changing 116 product landing pages and requires a lot of manual work
  • All business units need to restructure the information based on the new layout which can be resource intensive
  • Training to build new product pages
  • Apply the same changes to 6 other international domains which adds to the work of translation and rebuilt of the pages

Project Overview

The main goal was to improve the functionality of this page since it is one of the most visited pages on Quest/One Identity sites.

Following are some of the specific goals we had in mind to achieve:

  • Bring awareness to product lines
  • Add search functionality to products A-Z list
  • Improve navigation so that the users can access filters at anytime and update the values instantly
  • Improve filters functionality for better user experience

My Role

  • Create mockups based on research using Sketch app
  • Create prototype to communicate filters functionality with team using invision web prototype
  • Iterate on mockups using Frontify
  • Communicate functionality and hand off to developers using Zeplin

Issues

  • Navigation on page
  • No search functionality for product names
  • Ambiguity of product line to audience

Solution

  • Bring awareness to product lines by dedicating a section on this page to explain them
  • Adding a secondary nav and a back to top button to improve navigation
  • Add search functionality to quickly filter products list
  • Improve filters functionality so that they will be accessible at all times for update

Challenges

  • Work with stakeholders to finalize logos for each product line
  • Backend code modifications to update the way data is extracted for filters
  • Optimizing functionality and navigation of the page

Project/Request Planer

Sep 2017 |

Project Overview

Redesign existing project planning and ticketing system to facilitate project managers daily tasks. The main goal was to increase efficiency of planing work required for completing projects along with accurate SLAs.

My Role

  • Work with business analysts to understand depth of current system
  • List out user pain points
  • Create a new user journey map to show the flow
  • Introduce a new slick system to improve efficiency
  • Create mockups and prototype using Sketch app and Frontify
  • Work with developers to test and verify the system

Issues

  • lack of intuitiveness of the current system led to confusion for users
  • Creating a project was a long painful process
  • Users didn't know which type of request required for a certain work
  • Creating unnecessary and wrong requests resulted in inaccurate SLA estimation of projects

Solution

Design a more dynamic interface to eliminate user pain points and improve their daily tasks, break down information to easier to grasp pieces, and shed light to misleading terms and processes.

Challenges

  • Analysis to understand the depth of a complicated system
  • Numerous iterations on mock ups and workflow
