Quest Home Page Redesign

Aug 2017 |

Project Overview

The purpose of this project were to:

  • monitor user interactions using analytical tools
  • discover areas of improvement
  • conduct competitive analysis
  • align style with Quest branding
  • set up a split test to measure success
  • implement improvements based on test results
  • decrease page load time

My Role

  • Research and specify optimal image sizes for full-with responsive banner
  • Create templates for building new banners and share with creative team
  • Build banner guidelines on Quest brand library
  • Redesign solutions and featured products sections
  • Define interactions via prototypes
  • Testing
  • Set up split test using VWO

Issues

  • Old Dell style that didn't reflect our new Quest branding
  • Lack of effects and interactivity on page components
  • Page load time was quite high (7.85 seconds)
  • Duplicate information with no user engagement

Solution

After analysing data from heatmaps and Google analytics, we designed a variation of the current home page, applying brand guidelines and modern effects to enhance user experience.

Challenges

  • Accurate data analysis
  • Competitive analysis and put together a report
  • Train team members on responsive full-width banner design
  • Adapt with new technology (SVG & CSS3 effects)

UI Pattern Library

Oct 2016 |

Project Overview

Identify different UI components based on Atomic design concept and build a reusable library to provide cohesive user experience across Quest and its subsidiary sites.

My Role

  • Work with UI developers to create an uber spreadsheet, listing and categorizing UI elements into right buckets
  • Research Frontify brand management tool and train team members on how to use it
  • Study brand guidelines in industry and create taxonomy for our brand
  • Contribute building Quest web style guide and UI/UX pattern library
  • Create terms that are tangible for everyone in the company using Atomic design mental model
  • Build UX design library using Sketch app for design team, define a process to update library and train team members accordingly

Issues

  • Poor visibility on reusable web patterns
  • Lack of efficiency and maintainability of existing patterns
  • Inconsistent UI
  • Lack of resources for training new team members

Solution

Establish an effective interface design system to improve productivity and visibility for anyone touching the site, including designers, developers, copywriters, creative team & multimedia folks, project managers, and product owners.

Challenges

  • Plenty of research to make sure building it right and by standards
  • Train team members to create their dedicated sections
  • Lack of resources for maintenance

Research & Analysis

Uber spreadsheet listing out all UI elements based on Atomic design
Study brand guidelines in industry to create taxonomy for brand style guide and UI pattern library

Development

Contribute building Quest web style guide and UI/UX pattern library building tangible terminology
Build UX design library using Sketch app for design team

Project/Request Planer

Sep 2017 |

Project Overview

Redesign existing project planning and ticketing system to facilitate project managers daily tasks. The main goal was to increase efficiency of planing work required for completing projects along with accurate SLAs.

My Role

  • Work with business analysts to understand depth of current system
  • List out user pain points
  • Create a new user journey map to show the flow
  • Introduce a new slick system to improve efficiency
  • Create mockups and prototype using Sketch app and Frontify
  • Work with developers to test and verify the system

Issues

  • lack of intuitiveness of the current system led to confusion for users
  • Creating a project was a long painful process
  • Users didn't know which type of request required for a certain work
  • Creating unnecessary and wrong requests resulted in inaccurate SLA estimation of projects

Solution

Design a more dynamic interface to eliminate user pain points and improve their daily tasks, break down information to easier to grasp pieces, and shed light to misleading terms and processes.

Challenges

  • Analysis to understand the depth of a complicated system
  • Numerous iterations on mock ups and workflow

Design Process Overview

Iteration on Mocks

Use tabs for different steps involve in creating a project
Remove tabs and placing all tasks in one place to improve visibility. Use donut charts to show progress.

Final Mockup

Use vertical alignment of the cards to use real state better. Add user flexibility to add tasks missing.

Interactions

Drag and drop fuctionality to easily move tasks between phases
