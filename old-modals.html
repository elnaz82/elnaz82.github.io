Product Pages Redesign

Mar 2018 |

Project Overview

The goals of this project were to:

  • Apply functional minimalism rules
  • make product pages more dynamic and flexible to meet business needs
  • create a unique identity (theme) for individual business units
  • highlight key features of products to help improve user engagement
  • improve page navigation
  • create clear visual hierarchy and tell an appealing story about the product
  • conduct competitive analysis and add any information that can add value to show the benefits of using the product
  • align style with Quest branding
  • split test new layout to measure success
  • implement improvements based on test results

My Role

  • Design themes for 2 different business units using product-specific design patterns
  • Redesign sticky tabs
  • Design different options for a more engaging look for above the fold area
  • Design a section to showcase product benefits
  • Design different patterns to display key features of the product
  • Redesign product tour and improve interaction
  • Redesign product resources and develop a prototype to show functionality by hand coding Html, CSS3
  • Design a ribbon to show a quote from our customers (reviews)
  • Showcase customer logos
  • Design a ribbon to show relevant statistics if available
  • Design a ribbon to display awards and recognition of the product
  • Redesign social share
  • Conduct competitive analysis for any new component or concept (Quote, Stats, Awards, Benefits, Hero space, Tour, etc)

Issues

  • Text-heavy layout & lack of white space made the page hard to scan and comprehend
  • Key features of the products were buried in text and selling points can be easily missed by users
  • Lack of flexibility to modify the layout and add new sections
  • Lack of interactivity on page components
  • Not enough information to support the benefits of using the product
  • No uniques identity for different products

Solution

  • Visualize data using images or icons related to the content and get rid of heavy text to increase users focus
  • Focus o showing only the necessary information of the product based on people's preferences
  • Dynamic template concept to enable business units to create flexible content
  • Introduce new patterns to increase engagement and user experience
  • Visualize existing resources to support benefits (Quote, Awards, Stats, etc)
  • Increase the use of live demos and video content on the page
  • Design different themes based on product and give a unique look to each business unit
  • Apply design aesthetics without cluttering the layout (bigger titles, diagonal lines, abstract shapes, bright colors, etc.

Challenges

  • Updating product page template means changing 116 product landing pages and requires a lot of manual work
  • All business units need to restructure the information based on the new layout which can be resource intensive
  • Training to build new product pages
  • Apply the same changes to 6 other international domains which will add to the work of translation and rebuilt of the pages

Improve Registration Forms

May 2018 |

Project Overview

The main goals are:

  • Design a super short form to capture user's email and adjust display rules accordingly to get additional information based on new vs existing user
  • Conduct a research to improve layout and functionality
  • Speed up the registration process and improve conversion rate

My Role

  • Research best practices for form layout and validation
  • Conduct competitive analysis
  • Adjust functionality based on different types of registration if necessary
  • Create mockups showing all steps and communicate functionality with developers

Issues

  • The only way to capture leads were to demand a lot of data from users
  • Lack of instant validation for user error (inline validation)
  • Left align labels of form fields are problematic for international languages with longer text and leads to breaking form layout
  • No visual indication of required vs optional fields

Solution

  • Design a short form with only email field and show the longer form if the user is new
  • Conduct research/competitive analysis and come up with best practices for form functionality
  • Improve UI and functionality based on findings

Challenges

  • Quest forms are tied to several analytics tools and changing their functionality may require many complex updates in the backend
  • Comprehensive regression testing is required once the implementation is complete to make sure data is captured correctly
  • Monitor conversions accurately

Design Process Overview

Research

I started by reading several articles and research papers to find out the primary rules of designing an efficient form. Followings are some the best practices I captured:

  • Explicitly mark both required and optional fields
  • live inline validation has seen an explosive growth in adoption
  • The shorter the form, the better
  • Avoid defaulting fields to certain values unless you believe a large portion of your users will select that value
  • Use input masks where necessary
  • Auto-focus first form field in desktop to create an indication of where to start
  • Match keyboard to input type in mobile
  • Use autocomplete where possible to limit typing
  • Use sentence case for labels as they are easier to read
  • Top-aligned labels have the fastest completion time
  • Maintain inline label's placeholder text on focus. Floating labels is the way to go in this case
  • Avoid generic words for submit button
  • Avoid reset button and use the main CTA in the place that has the most convenient position for clicking in the viewport

Mockup

Based on my findings I have come up with the above solution including the following functionalities: (you can view the old form for reference)

  • Created a short form to capture users email address. If the user is existing, no more information will be required, otherwise, the second form will be presented to capture additional information
  • Used inline floaty labels to solve the issue with international localities and improve the speed of filling out the forms
  • Marked required and optional fields using *
  • Defined visual indicators for interactions (blue for focus, red for errors, green check marks for complete) to make the form more intuitive
  • Introduced inline live validation to
    • make locating errors easier
    • improve the speed of fixing errors while the field context is fresh in users minds
    • avoid skipping a required field unintentionally
  • Used sentence case labels
  • Used centered aligned orange CTA button for submission
